Introduction

Git is a distributed version control system created by Linus Torvalds in 2005.

It allows multiple people to work on a project simultaneously without interfering with each other's work.

Git is widely used in the software industry for source code management.

git --version
  • Distributed
  • Version Control
  • Collaboration

Tip: Always check your Git version to ensure compatibility with repositories you are cloning or collaborating on.

Getting Started

To start using Git, you need to install it on your computer.

After installation, configure your username and email:

Set your username:

git config --global user.name "Your Name"

Set your email:

git config --global user.email "you@example.com"
  • Installation
  • Configuration
  • Repository Initialization

Tip: Use the --global flag to set your username and email for all repositories. To set them for a specific repository, omit the --global flag.

Basic Commands

Here are some basic Git commands to get you started:

Initialize a new Git repository:

git init

This creates a new Git repository in the current directory.

Add files to the staging area:

git add .

This stages all the changes in the current directory for the next commit. To stage a specific file, use:

git add filename.txt

Commit changes:

git commit -m "Initial commit"

This commits your staged changes with a message describing what you have done.

  • git init
  • git add
  • git commit

Tip: Use descriptive commit messages to make it easier to understand the changes you made.

Branching

Branches in Git allow you to develop features and fix bugs in isolation.

Create a new branch:

git branch feature-branch

Switch to the new branch:

git checkout feature-branch

Or create and switch to a new branch in one command:

git checkout -b feature-branch

After creating a branch, you can work on your feature independently of the main codebase.

  • Creating Branches
  • Switching Branches
  • Branch Management

Tip: Use short and descriptive names for your branches to easily identify their purpose.

Merging

Merging combines changes from different branches into one.

Merge the feature branch into the main branch:

git checkout main git merge feature-branch

If there are any conflicts, Git will notify you, and you'll need to resolve them before completing the merge.

After resolving conflicts, use:

git add .

And then commit the merge:

git commit -m "Merged feature-branch into main"
  • Merge Command
  • Conflict Resolution
  • Merge Strategies

Tip: Regularly pull the latest changes from the main branch into your feature branch to minimize conflicts when merging.
